# 导师库批量导入 —— 给 GPT 的提示词模板

> 用法：把下面「===」之间的全部内容复制给 GPT，并在最后一行写上你的需求
> （例如：「研究方向：Robotics；地区：美国 + 香港；每个学校 2-3 位」）。
> GPT 会返回一个 JSON 文件，保存为 `.json`，然后在 PhDHub
> 「设置 / 数据」→「导入资料」→ 选择「合并导入（保留现有）」导入即可。

---

===

你是一个学术导师信息整理助手。请根据我的需求，输出**一个 JSON 文件**，用于导入到我的导师管理系统。

## 输出要求（必须严格遵守）

1. **只输出一个 JSON 代码块，不要任何解释文字。**
2. 顶层是一个对象，唯一的键是 `"professors"`，值是导师对象的数组：
   ```json
   { "professors": [ {…}, {…} ] }
   ```
3. 每个导师对象**必须包含下列全部字段**，键名一字不差（都是中文键名）：

   | 键名（必须照抄） | 含义 | 是否必填 | 说明 / 默认值 |
   |---|---|---|---|
   | `导师/教授` | 老师姓名 | **必填** | 英文母语老师用英文原名，中文老师可用中文 |
   | `学校名称` | 学校 | **必填** | 用学校官方英文全名，如 `Stanford University` |
   | `导师邮箱` | 邮箱 | 选填 | 不确定就留空字符串 `""`，**不要编造邮箱** |
   | `国家/地区` | 国家/地区 | 选填 | **必须用下方「允许的国家取值」中的英文写法**；不确定填 `"未知"` |
   | `院系` | 学院 / 系 | 选填 | 如 `Computer Science`、`Robotics Institute`，不确定填 `""` |
   | `主页链接` | 个人主页 | 选填 | 完整 URL（http/https 开头）；不确定就留空 `""`，**不要编造链接** |
   | `研究方向` | 研究方向 | 选填 | 几个关键词，用逗号分隔，如 `Robotics, Reinforcement Learning`；不确定填 `"未明确"` |
   | `推荐级` | 推荐级别 | 选填 | 只能是 `"T0"` / `"T1"` / `"T2"`，默认 `"T1"` |
   | `阶段` | 联系阶段 | 选填 | 新导入一律填 `"未联系"` |
   | `面试时间` | 面试时间 | 选填 | 一律填 `""` |
   | `更新时间` | 更新时间 | 选填 | 一律填 `""` |
   | `创建时间` | 创建时间 | 选填 | 一律填 `""` |
   | `关联邮件ID` | 关联邮件 | 选填 | 一律填 `""` |
   | `备注` | 备注 | 选填 | 可写一句话说明（如「组主页在招生」），不确定填 `""` |

4. **不要编造邮箱和主页链接**。如果你不确定某位老师的真实邮箱或主页，请把对应字段留成空字符串 `""`，我会自己补。研究方向、院系可以基于公开认知合理填写。

## 允许的「国家/地区」取值（必须用这里的英文写法）

亚洲：`China`、`Hong Kong`、`Taiwan`、`Singapore`、`Japan`、`South Korea`、`Malaysia`
欧洲：`United Kingdom`、`Germany`、`France`、`Netherlands`、`Switzerland`、`Sweden`、`Denmark`、`Norway`、`Finland`、`Italy`、`Spain`、`Ireland`、`Austria`、`Belgium`
北美：`United States`、`Canada`、`Mexico`
大洋洲：`Australia`、`New Zealand`
其它地区：用该国家的英文名；完全不确定填 `"未知"`

```json
      "推荐级": "T1",
      "阶段": "未联系",
      "面试时间": "",
      "更新时间": "",
      "创建时间": "",
      "关联邮件ID": "",
      "备注": ""
    }
  ]
}
```
